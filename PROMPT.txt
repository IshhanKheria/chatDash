Must have:
- WebSocket connection
- JWT auth for WebSocket connection
- Send direct messages in real time
- Receive direct messages in real time
- Persist each message to SQLite
- Handle receiver offline case by saving message anyway
- Acknowledge successful send

Suggested event flow:
- client connects with JWT
- server validates token and binds socket to user
- send message event
- message persisted
- sender gets confirmation
- receiver gets event if connected

Use clear event names. Example:
- connect / connected
- send_message
- receive_message
- message_sent
- error
